3. STATUS BADGES (Color-Coded)

```tsx
{/* Status Badge Helper Function */}
const getStatusBadge = (status: string) => {
  switch (status) {
    case 'success':
    case 'optimal':
      return (
        <Badge className="bg-green-500/10 text-green-600 border-green-500/30 text-[10px]">
          Success
        </Badge>
      );
    
    case 'error':
    case 'high':
      return (
        <Badge className="bg-red-500/10 text-red-600 border-red-500/30 text-[10px]">
          Error
        </Badge>
      );
    
    case 'warning':
    case 'low':
      return (
        <Badge className="bg-blue-500/10 text-blue-600 border-blue-500/30 text-[10px]">
          Warning
        </Badge>
      );
    
    case 'pending':
    case 'suboptimal':
      return (
        <Badge className="bg-amber-500/10 text-amber-600 border-amber-500/30 text-[10px]">
          Pending
        </Badge>
      );
    
    default:
      return (
        <Badge variant="outline" className="text-[10px]">
          N/A
        </Badge>
      );
  }
};

{/* Usage */}
<TableCell>{getStatusBadge(item.status)}</TableCell>
```

4. STATUS ICONS

```tsx
{/* Status Icon Helper */}
const getStatusIcon = (status: string) => {
  switch (status) {
    case 'success':
    case 'optimal':
      return <CheckCircle className="w-4 h-4 text-green-600" />;
    
    case 'error':
    case 'high':
      return <TrendingUp className="w-4 h-4 text-red-600" />;
    
    case 'warning':
    case 'low':
      return <TrendingDown className="w-4 h-4 text-blue-600" />;
    
    case 'pending':
    case 'suboptimal':
      return <AlertTriangle className="w-4 h-4 text-amber-600" />;
    
    default:
      return <HelpCircle className="w-4 h-4 text-gray-400" />;
  }
};

{/* Usage in Table */}
<div className="flex items-center gap-2">
  {getStatusIcon(item.status)}
  <span>{item.name}</span>
</div>
```

8. IMPLEMENTATION NOTES

Dependencies Needed:
```bash
npm install lucide-react
npm install motion/react
npm install sonner@2.0.3
```

Required Components (from shadcn/ui):
`Button`
`Card`
`Badge`
`Table` (TableHeader, TableBody, TableRow, TableHead, TableCell)
`Dialog` (DialogContent, DialogHeader, DialogTitle, DialogDescription)
`Alert` (AlertDescription)

Key Design Principles:

Consistent Color Coding
Green = Success/Good
Red = Error/High
Blue = Warning/Low
Amber = Pending/Needs Attention
Purple = Info/Metadata

Dark Mode Support
All colors have dark mode variants
Uses opacity for backgrounds (e.g., `/30`, `/20`)
Text uses `dark:text-*-400` for better contrast

Gradient Backgrounds
Cards use `from-{color}-50 to-white` in light mode
Dark mode uses `dark:from-{color}-950/30 dark:to-background`

Hover States
Table rows: `hover:bg-muted/50`
Clickable rows have specific hover colors per status

Typography
Headers: `text-2xl mb-1`
Metadata: `text-sm text-foreground/70`
Table text: `text-xs`
Card values: `text-xl`
